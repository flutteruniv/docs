### Issueの運用ルール
* ストーリー issue
  * Label は「story」に設定
  * 必ず Projects に紐付ける
  * 必ず担当者をアサインする
  * タスクを作ったら、説明欄にタスク issue の番号を記載する
    * Github issue には issue の親子を設定する機能が無いため運用でカバー
  * closeできるのはプロダクトオーナーのみ

* タスク issue
  * Label は「task」に設定
  * 初心者向けタスクの場合は「good first issue」もLabel に追加
  * Projects には紐付けない
  * タスクを作ったら、説明欄にストーリー issue の番号を記載する
    * Github issue には issue の親子を設定する機能が無いため運用でカバー
  * closeできるのはスクラムマスターのみ
